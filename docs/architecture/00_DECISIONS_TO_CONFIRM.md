# 00_DECISIONS_TO_CONFIRM

> 目的：在开发前把关键决策一次性拍板，避免后续返工与架构漂移。

## A. 连接与部署决策（最高优先级）

1. **Bridge 部署位置**
   - 决策项：Windows 本机常驻服务（推荐）
   - 说明：Visio COM 必须由 Windows 进程持有，WSL 不直接调用 COM。

2. **通信协议**
   - 决策项：HTTP/JSON（v1）
   - 备选：gRPC（后续可升级）
   - 说明：先保证可观测性与调试效率。

3. **监听地址与端口**
   - 决策项：默认仅本机/受控内网可达；固定端口（如 18761）
   - 说明：避免暴露到公网；便于脚本与运维。

4. **鉴权方案**
   - 决策项：静态 Bearer Token（v1），后续可升级短期令牌
   - 说明：最低安全门槛，防止误调用。

## B. 执行安全与稳定性决策

5. **COM 调度模型**
   - 决策项：单线程串行执行队列（STA）
   - 说明：禁止并发直打 COM，避免 Visio 崩溃与状态错乱。

6. **会话模型**
   - 决策项：显式 `session_id/document_id/page_id/shape_id`
   - 说明：禁用“当前选中对象”隐式语义。

7. **幂等机制**
   - 决策项：写操作强制 `request_id`
   - 说明：重试不重画，避免图污染。

8. **错误分级与重试策略**
   - 决策项：
     - 参数错误：不重试
     - 状态冲突：刷新状态后重试
     - 瞬态错误：有限重试
     - Visio 故障：触发恢复流程

9. **保护模式（熔断）**
   - 决策项：连续异常达到阈值后进入只读/拒绝写入模式
   - 说明：优先保护文档与 Visio 进程稳定。

## C. 可持续演进决策

10. **协议版本化**
    - 决策项：`tool_name@v1` 版本语义冻结，升级采用并行版本

11. **日志与审计最小字段**
    - 决策项：必须记录 `request_id, session_id, tool, params_hash, result, error_code, latency, timestamp`
    - 说明：为经验沉淀与回归测试提供基础数据。

12. **回归基准图集**
    - 决策项：建立固定任务集（如 two-box-arrow、resnet-block）用于每次升级回归

## D. 范围与节奏决策

13. **MVP 范围（非 demo）**
    - 单页图
    - 基础形状（矩形/菱形/文本）
    - 箭头连接
    - 对齐与间距
    - 基础样式（字体/线宽/颜色）
    - 失败恢复 + 日志

14. **暂缓项（避免过早复杂化）**
    - 多页协同编辑
    - 高级自动美化
    - 复杂图像理解
    - UI 自动化兜底（仅保留预案）

## E. 你需要确认的最终清单（勾选）

- [ ] v1 使用 HTTP + Bearer Token

- [ ] 启用 request_id 幂等
- [ ] 接受“先执行内核，后 drawskills 扩展”的节奏
