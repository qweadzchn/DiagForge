# Proposals

This directory stores structural improvement ideas that are not yet proven enough to become shared lessons or default repo behavior.

It is the place for:

- agent suggestions backed by real job evidence
- recurring operator friction
- missing controls or abstractions
- review mechanism improvements that still need validation

It is not the place for:

- raw round memory
- one-off complaints with no evidence
- final reusable lessons

## Promotion path

Use this directory as the middle stage:

`review -> proposal -> lesson or repo default`

## When to write a proposal

Write one when:

- the issue looks likely to repeat across jobs
- the owning layer is becoming clear
- the idea is promising but still needs another validation pass

## Suggested file naming

Use a date and a short slug:

- `2026-03-20-routing-intent-needs-better-readback.md`
- `2026-03-20-new-agent-needs-cold-start-smoke-test.md`

## Minimal proposal content

Use [proposal-template.md](proposal-template.md) and include:

- status
- the problem
- evidence
- suspected owning layer
- proposed change shape
- how to validate it
- when it should be promoted or rejected
